# EmailEvents Schema

**Table:** `EmailEvents`
**Source:** Microsoft Defender for Office 365 — via Microsoft 365 Defender connector
**Primary time field:** `Timestamp` (M365D Advanced Hunting table — NOT `TimeGenerated`)
**Schema source:** Inferred from training knowledge (Microsoft Learn returns 403 for WebFetch — verify with `EmailEvents | getschema`)
**Added:** 2026-05-24

---

## Column Reference

| Column | Type | Description |
|--------|------|-------------|
| `Timestamp` | datetime | Date and time the email was processed |
| `NetworkMessageId` | string | Unique identifier for the email generated by Microsoft 365 — **join key to UrlClickEvents, EmailUrlInfo, EmailAttachmentInfo** |
| `InternetMessageId` | string | Public-facing message ID from the email headers |
| `SenderMailFromAddress` | string | Sender email address from the MAIL FROM (envelope sender) — may differ from From header |
| `SenderFromAddress` | string | Sender email address in the From header |
| `SenderDisplayName` | string | Display name of the sender |
| `SenderObjectId` | string | Azure AD object ID of the sender (internal senders only) |
| `SenderIPv4` | string | IPv4 address of the originating mail server |
| `SenderIPv6` | string | IPv6 address of the originating mail server |
| `RecipientEmailAddress` | string | Recipient's email address — **primary filter for per-account investigation** |
| `RecipientObjectId` | string | Azure AD object ID of the recipient |
| `Subject` | string | Email subject line |
| `EmailDirection` | string | Direction: `Inbound`, `Outbound`, `Intra-org` |
| `DeliveryAction` | string | Outcome: `Delivered`, `Junked`, `Blocked`, `Replaced` |
| `DeliveryLocation` | string | Final delivery destination: `Inbox/folder`, `Junk folder`, `Quarantine`, `External`, `Failed`, `Dropped` |
| `ThreatTypes` | dynamic | Threat categories detected: `Phish`, `Malware`, `Spam`, `None` |
| `ThreatNames` | string | Specific threat names (malware family, campaign names) |
| `DetectionMethods` | dynamic | Detection methods applied: `Advanced filter`, `Antimalware engine`, `ATP detonation`, `URL reputation`, etc. |
| `ConfidenceLevel` | dynamic | Confidence levels associated with the threat types |
| `BulkComplaintLevel` | int | BCL (Bulk Complaint Level) score — higher = more likely bulk mail |
| `AttachmentCount` | int | Number of attachments in the email |
| `UrlCount` | int (stored as string in some queries — see note) | Number of URLs in the email body — **filter `UrlCount != "0"` to scope to emails that contained URLs** |
| `EmailActionPolicy` | string | Name of the policy that triggered the action |
| `EmailActionPolicyGuid` | string | GUID of the triggering policy |
| `AuthenticationDetails` | dynamic | SPF, DKIM, DMARC results |
| `OrgLevelAction` | string | Action taken at the organisation level |
| `OrgLevelPolicy` | string | Policy applied at the organisation level |
| `UserLevelAction` | string | Action taken based on per-user policy |
| `UserLevelPolicy` | string | Per-user policy applied |
| `ReportId` | long | Unique identifier for the event record |

## Critical Field Notes

### UrlCount — type ambiguity
The analyst-verified production query uses `UrlCount != "0"` (string comparison). KQL may implicitly cast integer fields to string for comparison. Use `!= "0"` as shown in the verified query — do NOT use `> 0` until confirmed in the environment.

```kql
// Verified production form:
| where UrlCount != "0"
```

### SenderMailFromAddress vs SenderFromAddress
- `SenderMailFromAddress` — the envelope sender (SMTP MAIL FROM) — used for SPF checking
- `SenderFromAddress` — the display address in the From header — what the user sees
- For phishing investigations, compare both; they often differ in spoofing attacks.

## Join Pattern (standard)

```kql
// EmailEvents → UrlClickEvents (email URL click investigation)
let EmailInformation = EmailEvents
    | where RecipientEmailAddress == "<target>"
    | where Timestamp > ago(2d)
    | where UrlCount != "0"
    | project Timestamp, NetworkMessageId, SenderMailFromAddress,
              SenderFromAddress, SenderDisplayName, ThreatNames;

EmailInformation
| join (
    UrlClickEvents
    | where ActionType != "ClickBlocked"
    | where Workload == "Email"
    | project Timestamp, Url, IPAddress, NetworkMessageId
  ) on NetworkMessageId
| sort by Timestamp desc
```

## Quick Lookup by Scenario

| Scenario | Filter |
|---|---|
| Emails received by specific account | `RecipientEmailAddress == "<upn>"` |
| Emails with URLs | `UrlCount != "0"` |
| Emails with attachments | `AttachmentCount > 0` |
| Phishing/malware deliveries only | `ThreatTypes has_any ("Phish","Malware")` |
| Inbound only | `EmailDirection == "Inbound"` |
| Successfully delivered | `DeliveryAction == "Delivered"` |
