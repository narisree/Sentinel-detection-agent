# EmailEvents Schema

**Table:** `EmailEvents`
**Source:** Microsoft Defender for Office 365 — via Microsoft 365 Defender connector
**Primary time field:** `TimeGenerated`
**Schema source:** Verified from MicrosoftDocs/azure-monitor-docs GitHub repository (2026-05-24)

## Column Reference (Live-Verified)

| Column | Type | Description |
|--------|------|-------------|
| `TimeGenerated` | datetime | Date and time (UTC) when the record was generated |

| `InternetMessageId` | string | Public-facing message ID from the email headers |
| `SenderMailFromAddress` | string | Envelope sender address from MAIL header (Return-Path) |
| `SenderMailFromDomain` | string | Envelope sender domain from MAIL header |
| `SenderFromAddress` | string | Sender address in the From header — visible to recipients |
| `SenderFromDomain` | string | Sender domain in the From header |
| `SenderDisplayName` | string | Display name of the sender as shown in the From header |
| `SenderObjectId` | string | Sender's Azure AD identifier |
| `SenderIPv4` | string | IPv4 address of the last detected mail server that relayed the message |
| `SenderIPv6` | string | IPv6 address of the last detected mail server |
| `RecipientEmailAddress` | string | Recipient's email address — **primary filter for per-account investigation** |
| `RecipientDomain` | string | Domain of the recipient |
| `RecipientObjectId` | string | Email recipient Azure AD identifier |
| `Subject` | string | Email subject line |
| `EmailDirection` | string | `Inbound`, `Outbound`, `Intra-org` |
| `DeliveryAction` | string | Action taken on the email |
| `DeliveryLocation` | string | Final storage: `Inbox/folder`, `Junk folder`, `Quarantine`, `Failed`, `Dropped`, `Deleted items` |
| `EmailAction` | string | Final action applied: junk, X-header, subject modification, redirect, delete, quarantine, or none |
| `EmailActionPolicy` | string | Policy type that triggered action: antispam, anti-phishing, antimalware, transport rules |
| `EmailActionPolicyGuid` | string | Unique identifier of the triggering policy |
| `LatestDeliveryAction` | string | Most recent action attempted on the message |
| `LatestDeliveryLocation` | string | Last known location of the email |

| `OrgLevelPolicy` | string | Organisational policy that triggered the action |
| `UserLevelAction` | string | Action taken based on per-user mailbox policy |
| `UserLevelPolicy` | string | Per-user mailbox policy applied |

| `ThreatNames` | string | Specific threat names detected in the message |
| `ThreatClassification` | string | Threat classification of the mail |

| `AdditionalFields` | dynamic | Additional information about the entity or event |
| `LastEventExecutionTime` | datetime | UTC timestamp when the record was last updated |
| `ReportId` | string | Unique identifier for the event |

## UrlCount — Type Note

`UrlCount` is `int`. Both filter forms work due to KQL implicit casting:

```kql
| where UrlCount > 0        // Preferred — correct int comparison
| where UrlCount != "0"     // Also works — production-verified form (KQL casts "0" to 0)
```

Use `> 0` for new queries. Accept `!= "0"` in analyst-provided queries — do not change it.

## Join Pattern (standard — KM-005)

```kql
let CompromizedEmailAddress = "";
let Timeframe = 2d;

let EmailInformation = EmailEvents
    | where TimeGenerated > ago(Timeframe)
    | where RecipientEmailAddress == CompromizedEmailAddress
    | where UrlCount > 0
    | project TimeGenerated, NetworkMessageId, SenderMailFromAddress,
              SenderFromAddress, SenderDisplayName, ThreatNames;

EmailInformation
| join (
    UrlClickEvents
    | where ActionType != "ClickBlocked"
    | where Workload == "Email"
    | project TimeGenerated, Url, IPAddress, NetworkMessageId
  ) on NetworkMessageId
| sort by TimeGenerated desc
```
